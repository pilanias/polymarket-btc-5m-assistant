## v1.1 Requirements (All Complete)

### Database Persistence

- [x] **DB-01**: Every trade insert and update is written to Supabase `trades` table in real-time
  - Outcome: `supabaseTradeStore.js` uses `upsert()` for all writes, called from `syncTradeToStore()`
- [x] **DB-02**: All trade read endpoints (`/api/trades`, `/api/paper/trades`, analytics, backtest) read from Supabase
  - Outcome: `getTradesFromStore()` made async, all 6 route handlers use `await`
- [x] **DB-03**: On first startup with an empty Supabase table, existing JSON ledger trades are migrated automatically
  - Outcome: `migrateFromLedger()` runs in `initTradeStore()` when `getTradeCount() === 0`
- [x] **DB-04**: When Supabase is unavailable (missing env vars or connection failure), system falls back to JSON ledger without crashing
  - Outcome: `initTradeStore()` catches errors, sets `_tradeStoreAvailable = false`, logs warning
- [x] **DB-05**: Supabase connection status is logged on startup (`[TradeStore] Supabase initialized` or fallback warning)
  - Outcome: Console log on success, `console.warn` on fallback

### Configuration

- [x] **CFG-01**: `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` documented in `.env.example` with setup instructions
  - Outcome: `.env.example` has full Supabase section with setup steps
- [x] **CFG-02**: `package.json` updated to add `@supabase/supabase-js` and remove `better-sqlite3`
  - Outcome: `@supabase/supabase-js: ^2.50.0` added, `better-sqlite3` removed
